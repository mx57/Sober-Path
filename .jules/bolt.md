## 2024-06-28 - [Rebranding and Stability]
**Инсайт:** Переименование приложения требует тщательного обновления всех конфигурационных файлов (app.json, package.json). Ошибки в зависимостях хуков (useCallback) приводят к критическим вылетам (crash) на нативных устройствах.
**Действие:** Всегда проверять массивы зависимостей хуков на наличие неопределенных переменных. При ребрендинге обновлять `slug` и `scheme` для корректной работы deep links.

## 2024-06-28 - [AI Coach Memory and Context]
**Инсайт:** ИИ-коуч становится более "живым", если использует накопленные в памяти (`ConversationMemory`) данные о достижениях пользователя. Простая подстановка строк из памяти значительно повышает вовлеченность.
**Действие:** Регулярно анализировать `userPreferences.goalsSet` в `AICoachService` для формирования персонализированных ответов.

## 2024-06-28 - [UI/UX: Loading States]
**Инсайт:** В сообществе и чате отсутствие индикации загрузки приводило к визуальным скачкам (layout shift).
**Действие:** Внедрять Skeleton-заглушки для всех асинхронных списков. Это улучшает восприятие скорости приложения (perceived performance).

## 2024-06-28 - [Data Persistence with AsyncStorage]
**Инсайт:** При использовании `Map` в сервисах для хранения состояния (например, `AICoachService.memory`), необходимо корректно преобразовывать их в объекты перед сериализацией в JSON для `AsyncStorage` (через `Object.fromEntries`).
**Действие:** Всегда использовать вспомогательные методы `loadFromStorage` и `saveToStorage` при работе с асинхронным хранилищем в синглтон-сервисах.

## 2024-06-28 - [AI Sentiment and Interactive Exercises]
**Инсайт:** Реакция ИИ на эмоциональный окрас (Sentiment) через простой маппинг ключевых слов эффективна для мобильных приложений без доступа к тяжелым LLM. Интерактивные упражнения типа «Рефрейминг» повышают ценность ассистента как терапевтического инструмента.
**Действие:** Использовать `detectSentiment` для адаптации тона ответов. Внедрять пошаговые упражнения через состояние `activeExercise` в ассистенте.

## 2024-06-28 - [Performance: Reanimated for Progress Bars]
**Инсайт:** Стандартный `width` в `StyleSheet` при анимации через `state` вызывает лишние ре-рендеры компонента.
**Действие:** Использовать `useAnimatedStyle` и `withTiming` из `react-native-reanimated` для плавных переходов прогресс-баров без нагрузки на JS-поток.
